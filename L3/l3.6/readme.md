## Файлы для SalesTracker — CRUD с аналитикой и агрегированием данных  

### 📋 Описание проекта  

Во многих сервисах нужно не просто хранить записи (например, транзакции, заказы, события),  
но и уметь анализировать их: смотреть сумму продаж за период, количество заказов в день,  
средний чек, медиану, перцентиль и т.п. Это может быть полезно в личных финансовых приложениях,  
системах учёта заказов, внутренних CRM-инструментах и многих других случаях.  

В этом проекте реализован сервис, который умеет не только выполнять CRUD-операции,  
но и собирать по этим данным аналитику.  
Сервис предоставляет:  
- CRUD-операции для записей (продажи/расходы);  
- агрегированную аналитику за произвольный период (сумма, среднее, количество, медиана, 90‑й перцентиль);  
- группировку данных по категориям;  
- экспорт данных в CSV;  
- простой веб‑интерфейс для удобного взаимодействия.  

### 🖥️ Возможности  

Встроенные HTTP-методы:  

  - **POST /items** — создание новой записи (категория, сумма, дата);  
  - **GET /items** — получение списка записей за период с возможностью сортировки по полям (date, category, amount);  
  - **PUT /items/{id}** — обновление существующей записи;  
  - **DELETE /items/{id}** — удаление записи.  

Аналитические эндпоинты:  

  - **GET /analytics?from=...&to=...** — список всех предстоящих мероприятий;  
  - **GET /analytics/by-category?from=...&to=...** — группировка по категориям (сумма и количество в каждой категории).  

Дополнительные возможности:  

- **GET /export/csv?from=...&to=...** — экспорт записей за период в формате CSV.  
- Простой веб-интерфейс, позволяющий:  
  - добавлять, редактировать и удалять записи;  
  - просматривать таблицу записей с фильтром по дате;  
  - видеть сводную аналитику (метрики);  
  - скачивать CSV-отчёт;  
  - включать/скрывать группировку по категориям,  
  - сортировать записи по клику на заголовки таблицы.  

### 🗂️ Структура проекта  

```bash
├── cmd/
```

### 🚀 Быстрый старт  

**Требования:**  

- Docker  
- Свободные порты:  

    8081 — веб-интерфейс и API  
    5432 — PostgreSQL  

**Docker-образы**  

- postgres:16  

**Запуск системы:**  

    cd ./ && docker compose up  


- После успешного запуска веб-интерфейс будет доступен по адресу: http://localhost:8081  

### ⚙️ Конфигурация  

Все настройки задаются через файл .env в корне проекта.

    ## переменные сервера и api
    SERVICE_HOST_NAME=0.0.0.0          # хост, на котором слушает сервис
    SERVICE_PORT=8081                  # порт сервера
    GIN_MODE=debug                     # режим логов Gin (debug / release)

    ## переменные базы данных
    DB_HOST_NAME=postgres              # имя контейнера PostgreSQL в сети Docker
    DB_PORT=5432                       # порт PostgreSQL
    DB_NAME=postgres                   # имя базы данных
    DB_PASSWORD=postgres               # пароль
    DB_USER=postgres                   # пользователь
