## Файлы для ImageProcessor — асинхронная обработка изображений с очередями и S3-хранилищем  

### 📋 Описание проекта  

Сервисы, где пользователи загружают изображения, встречаются повсюду: соцсети, маркетплейсы,  
фотогалереи, блоги, редакторы и т.д. Чтобы изображения отображались корректно и быстро — под  
каждую задачу нужны свои версии: миниатюры, уменьшенные копии, варианты с водяными знаками и пр.  
Обрабатывать их синхронно — долго и неудобно, особенно при высокой нагрузке.  

В этом проекте реализован сервис, который принимает изображения, кладёт задачу на обработку  
в очередь (через Apache Kafka), и уже в фоне обрабатывает файл (например, делает resize или  
ставит watermark). Такая архитектура позволяет не блокировать пользователя и обрабатывать  
изображения параллельно, масштабируемо и гибко.  

Пользователь загружает файл через веб-интерфейс, указывает нужные операции (ресайз, миниатюра,  
водяной знак) и мгновенно получает UUID задачи.  
Дальнейшая обработка происходит в фоне:  

- менеджер сохраняет оригинал в MinIO (S3-совместимое хранилище),  
- записывает задачу в outbox-таблицу PostgreSQL,  
- специальный релей отправляет задачи в Kafka,  
- воркеры (processor) читают топик, скачивают файл, применяют операции и загружают результаты  
обратно в MinIO,  
- финальный результат отправляется в выходной топик, и менеджер обновляет статус в БД.  

Готовые изображения можно скачать через веб-интерфейс или API. Вся метаинформация хранится в  
PostgreSQL, сами файлы — в MinIO. Такой подход обеспечивает отказоустойчивость,  
масштабируемость и не блокирует пользователя.  

### 🖥️ Возможности  

Встроенные HTTP-методы (реализованы в менеджере):  

  – **POST /upload** — загрузка изображения с параметрами обработки;  
  – **GET /image/:id** — получение обработанного варианта изображения;  
  – **DELETE /image/:id** — удаление изображения и всех его вариантов;  
  – **GET /images?limit=N** — список последних загруженных оригиналов с их вариантами (для галереи).  

Дополнительно:  
- асинхронная обработка через Kafka с гарантией доставки (outbox pattern);  
- хранение файлов в S3 (MinIO);  
- возможность одновременной работы нескольких воркеров;  
- простой и понятный UI на чистом HTML/JS.  

### 🗂️ Структура проекта  

```bash
├── cmd/
│   ├── manager/            # точка входа для менеджера (HTTP + outbox relay + consumer результатов)
│   └── processor/          # точка входа для воркера (consumer задач + producer результатов)
├── pkg/
│   ├── configuration/      # загрузка конфигурации из .env
│   ├── domain/             # общие доменные модели (ImageData, Task, Result и т.д.)
│   ├── manager/
│   │   ├── api/            # обработчики HTTP и модели ответов
│   │   ├── db/             # работа с PostgreSQL (таблицы images, outbox; миграции)
│   │   ├── server/         # запуск HTTP-сервера (Gin + middleware)
│   │   └── service/        # бизнес-логика менеджера (загрузка, получение, удаление)
│   ├── processor/
│   │   ├── operations/     # функции обработки изображений (resize, thumbnail, watermark)
│   │   ├── worker/         # воркер, который скачивает из S3, обрабатывает и загружает обратно
│   │   └── models/         # копии доменных моделей для воркера
│   └── s3/                 # клиент для MinIO (загрузка, скачивание, удаление, создание бакета)
├── web/                    # статические файлы веб-интерфейса (index.html)
├── compose.yml             # оркестрация всех сервисов (менеджер, процессор, postgres, kafka, minio)
├── Dockerfile.manager      # сборка образа менеджера
├── Dockerfile.processor    # сборка образа процессора
├── .env                    # переменные окружения (пример)
├── go.mod, go.sum          # зависимости Go
└── readme.md               # этот файл
```

### 🚀 Быстрый старт  

**Требования:**  

- Docker  
- Свободные порты:  

    8081 — веб-интерфейс и API  
    5432 — PostgreSQL  
    9092 — Kafka  
    9000 — MinIO (S3 API)  
    9001 — консоль MinIO  

**Docker-образы**  

- postgres:16  
- redis:7  

**Запуск системы:**  

    cd ./ && docker compose up  


- После успешного запуска веб-интерфейс будет доступен по адресу: http://localhost:8081  

### ⚙️ Конфигурация  

Все настройки задаются через файл .env в корне проекта.

    ## переменные сервера и api
    SERVICE_HOST_NAME=0.0.0.0         # хост, на котором слушает менеджер
    SERVICE_PORT=8081                 # порт сервера
    GIN_MODE=debug                    # режим логов Gin (debug / release)

    ## переменные сервиса
    WIDTH_THUMBNAIL=150        # ширина миниатюры по умолчанию
    HEIGHT_THUMBNAIL=200       # высота миниатюры по умолчанию

    ## переменные базы данных
    DB_HOST_NAME=postgres           # имя контейнера PostgreSQL в сети Docker
    DB_PORT=5432                    # порт PostgreSQL
    DB_NAME=postgres                # имя базы данных
    DB_PASSWORD=postgres            # пароль
    DB_USER=postgres                # пользователь

    ## переменные S3 хранилища (MinIO)
    S3_ENDPOINT=minio:9000                # эндпоинт MinIO (внутри Docker)
    S3_ACCESS_KEY=minioadmin              # ключ доступа
    S3_SECRET_KEY=minioadmin              # секретный ключ
    S3_BUCKET=images                      # имя бакета для изображений
    S3_USE_SSL=false                      # использовать SSL? (для локальной разработки false)

    ## переменные Kafka
    KAFKA_PORT_NUM=9092                           # порт Kafka
    KAFKA_HOST_NAME=kafka                         # имя хоста Kafka в Docker
    KAFKA_BROKERS=kafka:9092                      # строка подключения (для совместимости)
    KAFKA_INPUT_TOPIC=image-tasks                 # топик для задач на обработку
    KAFKA_OUTPUT_TOPIC=image-results              # топик для результатов
    KAFKA_CONSUMER_GROUP=image-processor-manager  # группа потребителей для менеджера
