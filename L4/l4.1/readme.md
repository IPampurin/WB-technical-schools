## Файлы для MergeDoneChannels — пакет объединения done-каналов  

### 📋 Описание проекта  

Проект представляет собой пакет 'or', экспортирующий функцию объединения done-каналов.  
Проект хоть и небольшой, но концептуально важен: правильная работа с конкурентностью и каналами.  

### 🖥️ Возможности  

Функция or.Or принимает произвольное количество каналов типа <-chan interface{} и возвращает  
новый канал, который закроется, как только закроется любой из входных. Это удобно для  
объединения сигналов завершения (done‑каналов) в конкурентных программах.  
Подробный пример с выводом находится в файле example_test.go.  

- Реализация функции Or использует sync.Once для гарантии однократного закрытия выходного канала.  
Обрабатывается случай отсутствия каналов.  
- Тесты проверяют все ключевые сценарии: отсутствие каналов, один канал, несколько каналов,  
наличие уже закрытого канала, а также простую проверку на утечки горутин (хотя полная проверка  
требует более сложных инструментов).  
- Пример демонстрирует использование функции с некоторым набором каналов.  
Вывод в комментарии "// Output: done after 1s" - не строгий, а приблизительный, так как точное  
время может варьироваться.

### 🚀 Быстрый старт  

Установка пакета:  

    go get github.com/IPampurin/MergeDoneChannels/or  

Использование в коде:  

    go
    package main

    import (
        "fmt"
        "time"
        "github.com/IPampurin/MergeDoneChannels/or"
    )

    func main() {
        // вспомогательная функция, создающая канал, который закроется через заданное время
        sig := func(after time.Duration) <-chan interface{} {
            c := make(chan interface{})
            go func() {
                defer close(c)
                time.Sleep(after)
           }()
           return c
        }

        start := time.Now()
        <-or.Or(
            sig(2*time.Hour),
            sig(5*time.Minute),
            sig(1*time.Second),
            sig(1*time.Hour),
            sig(1*time.Minute),
        )
        fmt.Printf("done after %v\n", time.Since(start).Round(time.Second))
        // Output: done after 1s
    }

### 🧪 Тестирование  

Для запуска тестов выполните в корне пакета:  

    go test -v

Тесты проверяют:  
- пустой список каналов (возвращается закрытый канал);  
- один канал;  
- несколько каналов;  
- наличие уже закрытого канала;  
- отсутствие утечек горутин (базовая проверка).
