## Файлы для EventCalendar — сервис календаря с напоминаниями и архивацией  

### 📋 Описание проекта  

**EventCalendar** — это бэкенд-сервис для управления событиями пользователей с возможностью автоматических напоминаний.  
Пользователи создают события (встречи, задачи, дедлайны) с произвольной длительностью, указывают дату и время напоминания.  
Сервис самостоятельно отслеживает наступление этого момента и отправляет уведомление (логирует, можно расширить до email/telegram).  

Старые события автоматически перемещаются в архив, не засоряя основную таблицу.  
Все операции логируются асинхронно, а напоминания обрабатываются через очередь с приоритетом - это гарантирует точность и устойчивость при перезапуске.  

Сервис подходит для CRM, корпоративных порталов, личных планировщиков и любых систем, где требуется управление временными событиями.  

**EventCalendar** - готовый к использованию микросервис с чистой архитектурой, асинхронными воркерами и простым веб-интерфейсом.  
Может быть расширен для реальной отправки уведомлений (SMTP, Telegram, Webhook) без изменения основной логики.  

### 🖥️ Возможности  

**Управление событиями (CRUD):**  
- Создание события с заголовком, описанием, временем начала/окончания, опциональным напоминанием.
- Обновление и удаление событий с автоматическим перепланированием напоминаний.
- Получение событий по ID, за день, неделю, месяц (с учётом часового пояса).

**Напоминания:**  
- Асинхронный планировщик с приоритетной очередью (время отправки → событие).
- Восстановление напоминаний из БД после перезапуска.
- Отмена напоминаний при удалении или изменении события.

**Архивация:**  
- Фоновая горутина, которая каждую минуту (настраивается) переносит прошедшие события в архивную таблицу.
- Пагинированный просмотр архива по пользователю.

**Инфраструктура:**  
- PostgreSQL для хранения активных и архивных событий (миграции при старте).
- Асинхронный логгер с буферизацией (не блокирует основной поток).
- Graceful shutdown HTTP-сервера и всех фоновых воркеров.
- Простой веб-интерфейс для тестирования (HTML+JS).
- Docker Compose для быстрого развёртывания.  

### 📡 HTTP-методы (API)  

  - **POST /create_event** — создание нового события (название, описание, время начала/окончания, опциональное напоминание);  
  - **POST /update_event** — обновление существующего события (изменение любых полей, перепланирование напоминания);  
  - **POST /delete_even**t** — удаление события (отмена запланированного напоминания, если было);  
  - **GET /events_for_day?user_id={id}&date={YYYY-MM-DD}** — список активных событий пользователя на указанный календарный день (с учётом часового пояса);  
  - **GET /events_for_week?user_id={id}&date={YYYY-MM-DD}** — список событий на неделю (понедельник–воскресенье), содержащую указанную дату;  
  - **GET /events_for_month?user_id={id}&date={YYYY-MM-DD}** — список событий на календарный месяц указанной даты;  
  - **GET /archive_events?user_id={id}&limit={n}&offset={m}** — получение архивных событий пользователя с пагинацией (limit по умолчанию 50).

``` bash
│   ├── service/                # бизнес-логика, интерфейсы (репозиторий, логгер, планировщик)
│   ├── storage/
│   │   └── sqlDB/              # реализация репозитория на PostgreSQL (миграции, запросы)
│   └── transport/
│       └── http/               # Gin-сервер, хендлеры, DTO, middleware, маршруты
├── web/                        # статика (index.html, style.css, script.js)
├── tests/                      # модульные и интеграционные тесты
├── .env                        # пример переменных окружения
├── compose.yml                 # запуск app + postgres
├── Dockerfile                  # multi-stage сборка
├── go.mod, go.sum
└── readme.md
```

### 🚀 Быстрый старт  

**Требования:**  
- Docker и Docker Compose  
- Свободные порты: 8081 (HTTP), 5432 (PostgreSQL)  

**Запуск:**  

    docker compose up --build

**После успешного запуска:**  

    http://localhost:8081

API доступно по тем же портам (см. раздел «Возможности»).  

Остановка:  

    docker compose down

### ⚙️ Конфигурация  

Все настройки задаются через файл .env в корне проекта.  

    HTTP_HOST=0.0.0.0            # имя службы (контейнера) в докере
    HTTP_PORT=8081               # порт хоста, на котором работает сервис
    HTTP_READ_TIMEOUT=15s        # таймаут чтения тела запроса
    HTTP_WRITE_TIMEOUT=15s       # таймаут записи ответа
    HTTP_IDLE_TIMEOUT=60s        # keep-alive
    HTTP_SHUTDOWN_TIMEOUT=30s    # ожидание завершения при graceful shutdown
    
    DB_HOST=postgres         # имя службы (контейнера) в докере
    DB_PORT=5432             # порт, на котором сидит база данных
    DB_NAME=postgres         # имя базы данных
    DB_PASSWORD=postgres     # пароль базы данных
    DB_USER=postgres         # имя пользователя базы данных

    APP_TIMEZONE=Europe/Moscow             # временная зона по умолчанию
    APP_ARCHIVE_EVERY=1m                   # периодичность проверки архиватора

    APP_REMINDER_QUEUE_SIZE=10             # буфер канала напоминальщика
    APP_LOG_BUFFER_SIZE=10                 # буфер канала асинхронного логгера

### 🧪 Тестирование  

Запуск всех тестов, включая интеграционные (необходим запущенный docker):  

    go test ./tests -v

Тесты покрывают:  

- бизнес-логику CalendarService  
- HTTP-обработчики  
- работу планировщика напоминаний  
- архиватор  
- асинхронный логгер  
- репозиторий (PostgreSQL)  

### 📦 Зависимости  

**Gin** – HTTP-роутер и middleware  
**pgx** – драйвер PostgreSQL  
**google/uuid** – генерация идентификаторов  
**testify** – assertions и моки для тестов  
**testcontainers** – интеграционные тесты с реальной БД  

Все зависимости управляются через go.mod.
